Export a detailed CSV of every transaction in a payout batch. The export includes amounts, fees, exchange rates, and customer details — everything your finance team needs to reconcile payouts against your accounting records.

## Exporting from the console

<Steps>
<Step title="Open the payout detail page">
  Navigate to **Payouts** in the merchant console and click the batch you want to export.
</Step>

<Step title="Click Export CSV">
  Click the **Export CSV** button on the detail page. The download starts immediately.
</Step>
</Steps>

The file is named `payout-{payment_reference}-transactions.csv` using the batch's payment reference.

## Exporting via API

You can also export programmatically using the API.

### Prerequisites

- A Quidkey merchant account with `client_id` and `client_secret`
- An active access token (see [Authentication](/api-reference/introduction#authentication))
- The `payout_batch_id` of the batch to export

### Request

<CodeGroup>

```bash cURL
curl -X GET 'https://core.quidkey.com/api/v1/merchants/{merchantId}/payouts/{payoutBatchId}/transactions/export' \
  -H 'Authorization: Bearer YOUR_ACCESS_TOKEN' \
  -o transactions.csv
```

```javascript Node.js
const response = await fetch(
  `https://core.quidkey.com/api/v1/merchants/${merchantId}/payouts/${payoutBatchId}/transactions/export`,
  {
    headers: {
      'Authorization': `Bearer ${accessToken}`
    }
  }
);

const csv = await response.text();
```

```python Python
import requests

response = requests.get(
    f'https://core.quidkey.com/api/v1/merchants/{merchant_id}/payouts/{payout_batch_id}/transactions/export',
    headers={'Authorization': f'Bearer {access_token}'}
)

csv_content = response.text
```

</CodeGroup>

The response is a `text/csv` file with a `Content-Disposition` header for the filename.

## CSV field reference

Each row in the CSV represents one transaction in the batch.

| Column | Description | Example |
|--------|-------------|---------|
| **Transaction Number** | Unique transaction identifier | `TXN-20240401-001` |
| **Type** | `payment` or `refund` | `payment` |
| **Status** | Transaction status | `completed` |
| **Amount** | Transaction amount in the original currency | `50.00` |
| **Currency** | Original transaction currency (ISO 4217) | `EUR` |
| **Converted Amount** | Amount in the payout currency after conversion | `43.50` |
| **Payout Currency** | Currency of the payout batch | `GBP` |
| **Exchange Rate** | Conversion rate applied (empty if same currency) | `0.87` |
| **Fee Total** | Total fees deducted from this transaction | `1.25` |
| **Fee Currency** | Currency of the fees | `GBP` |
| **Fee Details** | Breakdown of individual fees, pipe-separated | `processing:0.75\|scheme:0.50` |
| **Customer Email** | Customer's email address (if provided) | `jane@example.com` |
| **Order ID** | Your internal order reference (if provided at payment creation) | `ORD-12345` |
| **Created At** | When the transaction was created (ISO 8601) | `2024-04-01T10:30:00Z` |
| **Paid Out At** | When the payout was executed (ISO 8601) | `2024-04-05T14:00:00Z` |

<Note>
**Amounts are in decimal format.** The CSV converts minor units to decimal for readability — `5000` cents becomes `50.00`. This differs from the API, which uses minor units.
</Note>

## Understanding fees

The **Fee Total** column shows the sum of all fees for a transaction. The **Fee Details** column provides the breakdown, formatted as `type:amount` pairs separated by pipes (`|`).

For example, a fee details value of `processing:0.75|scheme:0.50` means:

| Fee type | Amount |
|----------|--------|
| Processing | 0.75 |
| Scheme | 0.50 |
| **Total** | **1.25** |

## Tips for reconciliation

<AccordionGroup>
<Accordion title="Matching payouts to bank statements">
  Use the **payment reference** from the payout batch detail page. This reference appears on your bank statement and uniquely identifies the payout.
</Accordion>

<Accordion title="Linking transactions to orders">
  The **Order ID** column maps to the `order_id` you provided when creating the payment request. Use this to match CSV rows to your invoices or order management system.
</Accordion>

<Accordion title="Handling cross-currency payouts">
  For cross-border payments, the **Amount** and **Currency** columns show the original transaction values, while **Converted Amount** and **Payout Currency** show what was actually paid out. The **Exchange Rate** column shows the conversion rate applied.
</Accordion>

<Accordion title="Importing into accounting software">
  The CSV is compatible with standard accounting imports (Xero, QuickBooks, Excel). The decimal amounts and ISO date formats are designed for direct import without transformation.
</Accordion>
</AccordionGroup>
